# Team OS — Manual команды 149

Интерактивный лендинг-инструкция по применению команды 149 (Спринт 5 «Я понимаю себя»).

Одностраничное приложение с клиентской навигацией:

- **Главная** — hero, метрики, цикл из 5 спринтов, карточки участников, интерактивная карта «Найти коллегу», принципы работы, переключаемые режимы, риск-протокол, CTA.
- **Персональные страницы** 4 участниц (Анна К., Лиза Б., Алиса Ч., Яна К.) — data-driven, 11 секций, sticky subnav, reset-протокол.
- **Инструкция новому участнику** — onboarding-гайд с ролями, процессом, правилами общения и принципами.
- **Игра «Найди подходящего коллегу»** — дерево решений Да/Нет с рекомендацией участника.

## Файлы в этом репозитории

```
index.html    — главная точка входа, весь сайт (шрифты и скрипты подключены)
style.css     — базовые стили: сброс, шрифты, keyframe-анимации
404.html      — копия index.html, чтобы прямые ссылки не показывали "404" GitHub
.nojekyll     — отключает обработку Jekyll на GitHub Pages (обязательно!)
README.md     — этот файл
```

Открой `index.html` двойным кликом — сайт запустится локально без сервера.

## Публикация на GitHub Pages — пошагово

1. Создай **новый репозиторий** на GitHub (публичный — приватные репозитории поддерживают Pages только на платных планах).
2. Залей **всё содержимое этой папки прямо в корень репозитория** (не в подпапку!). То есть после загрузки в корне репозитория сразу должны лежать `index.html`, `style.css`, `404.html`, `.nojekyll`, `README.md`.
3. Зайди в репозиторий → **Settings → Pages**.
4. В разделе **Build and deployment → Source** выбери **Deploy from a branch**.
5. **Branch**: выбери `main` (или ту ветку, куда залил файлы) и папку **/ (root)** → **Save**.
6. Подожди 1–2 минуты — вверху появится зелёная плашка со ссылкой вида:
   `https://<твой-username>.github.io/<название-репозитория>/`
7. Открой именно эту ссылку.

## Если видишь 404

Почти всегда причина одна из этих:

- **Файлы лежат не в корне репозитория**, а в подпапке (например `github-pages/index.html` вместо `index.html`). GitHub Pages ищет `index.html` строго в корне выбранной папки (root или `/docs`).
- **Pages ещё не включены** — зайди в Settings → Pages и проверь, что Source настроен и появилась зелёная ссылка "Your site is live at...".
- **Названо не `index.html`** — обрати внимание на регистр и точное имя файла.
- **Открыл не ту ссылку** — адрес обязательно содержит имя репозитория:
  `https://username.github.io/repo-name/`, а не просто `https://username.github.io/`.
- **Только что задеплоил** — подожди 1–2 минуты, GitHub Pages собирается не мгновенно (смотри вкладку Actions — там виден прогресс сборки).
- **Ветка не та** — если пушишь в `master`, а в настройках Pages выбран `main` (или наоборот), Pages не увидит файлы.

Если после проверки всех пунктов всё ещё 404 — пришли ссылку на репозиторий, посмотрю, что не так.
